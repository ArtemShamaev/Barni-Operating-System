# BarniOS Professional Edition

**BarniOS** – это минималистичная, но мощная 16-битная операционная система, написанная на ассемблере x86. Она включает виртуальную файловую систему в ОЗУ, текстовый редактор, игры, калькулятор и командную строку.

## 🚀 Возможности

- **Виртуальная файловая система в ОЗУ** – до 16 файлов/директорий, каждый до 1024 байт.
- **Полноценный текстовый редактор** с поддержкой навигации курсором, вставки/удаления, Page Up/Down.
- **Командная строка** с динамическим приглашением (текущий путь).
- **Поддержка директорий** (имена с суффиксом `/`) и команд `cd`, `cd ..`, `mkdir`.
- **Игра «Угадай число»** и **калькулятор** (+, -, *, /).
- **Отображение системного времени и даты** (через BIOS).
- **Графический интерфейс** с рамкой, меню, строкой состояния.
- **Кастомизация** – имя пользователя и пароль задаются через `make config`.

## 🔧 Установка и запуск

### Требования
- Утилита `make`
- Ассемблер `NASM`
- Эмулятор `QEMU` (рекомендуется) или реальное оборудование (загрузка с USB/дискеты)

### Windows

1. Установите `make` через Chocolatey (запустите PowerShell от имени администратора):
   ```powershell
   Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
   choco install make

Установите NASM и QEMU вручную или через Chocolatey:
powershell

choco install nasm qemu

Скачайте исходный код BarniOS.

В командной строке (или PowerShell) перейдите в папку проекта и выполните:
cmd

make config
make compile
make run

Linux (Ubuntu/Debian)
bash

sudo apt update
sudo apt install make nasm qemu-system-x86
make config
make compile
make run

Arch Linux
bash

sudo pacman -S make nasm qemu-full
make config
make compile
make run

Mandriva / OpenMandriva
bash

sudo urpmi make nasm qemu
make config
make compile
make run

    Примечание: Цель make download автоматически установит зависимости (NASM, QEMU) в большинстве дистрибутивов.

🛠 Настройка имени пользователя и пароля

Перед первой сборкой необходимо задать учётные данные:
bash

make config

Скрипт запросит имя пользователя и пароль. Пароль будет запрашиваться при каждой загрузке системы перед запуском ядра.
📟 Команды BarniOS

После загрузки и успешного ввода пароля вы попадёте в главное окно с синей рамкой и строкой состояния. Внизу (строка 22) находится приглашение с текущим путём (например C:\>).
Работа с файлами и директориями
Команда	Описание	Пример
ls	Показать список файлов и директорий в текущей папке.	ls
write to <имя>	Создать новый файл и открыть его в редакторе.	write to notes.txt
write in <имя>	Открыть существующий файл для редактирования.	write in docs.txt
mkdir <имя>	Создать директорию (имя автоматически дополняется /).	mkdir myfolder
cd <имя>	Перейти в указанную директорию (поддиректорию).	cd myfolder
cd ..	Перейти на уровень выше (в родительскую директорию).	cd ..

Особенности:

    Имена файлов и директорий не должны содержать пробелы.

    Директории визуально помечаются как [DIR] имя.

    Максимальный размер файла – 1024 байта.

Системные команды
Команда	Описание
clr	Очистить экран и перерисовать интерфейс.
reboot	Перезагрузить систему.
help	Показать полный список команд с кратким описанием.
sysinfo	Вывести ASCII-логотип и информацию об авторских правах.
time	Показать текущее системное время (часы:минуты).
date	Показать текущую дату (дд.мм.гггг).
Игры и приложения
Команда	Описание
guess	Запустить игру «Угадай число» (0–100).
calc	Калькулятор с операциями +, -, *, / (целые числа до 65535).
✍️ Текстовый редактор

При создании или открытии файла (write to / write in) вы попадаете в полноэкранный редактор.

Управление:

    Клавиши со стрелками – перемещение курсора.

    Backspace – удаление символа слева.

    Enter – перевод строки.

    Page Up / Page Down – прокрутка по страницам.

    ESC – выход в командную строку (изменения автоматически сохраняются).

Особенности:

    Вверху отображается имя файла и кнопки (минимизация/закрытие пока декоративны).

    Внизу строка состояния: имя файла, позиция (Ln:Col), размер в байтах, признак [READ ONLY] для системных файлов (например docs.txt).

    Редактор поддерживает файлы до 1024 байт.

📁 Файловая система

    Все файлы и директории хранятся в оперативной памяти (виртуальная ФС).

    Каждый файл описывается структурой: имя (32 байта), размер (2 байта), данные (1024 байта).

    Директории – это файлы, у которых последний символ имени – /. При создании через mkdir слеш добавляется автоматически.

    Корневая директория – C:\.

    При загрузке системы автоматически создаётся файл docs.txt (справка, read-only).

Ограничения:

    Максимум 16 элементов (файлов + папок).

    Длина имени – до 31 символа (плюс завершающий ноль).

📂 Структура проекта
text

.
├── boot.asm          # Загрузчик (сектор 1) – запрос пароля, загрузка ядра
├── kernel.asm        # Ядро ОС (сектора 2-100) – команды, ФС, редактор
├── Makefile          # Сборка, настройка, запуск
├── config.sh         # Скрипт для ввода имени пользователя и пароля
├── boot_config.inc   # Автоматически генерируется make (имя/пароль)
└── README.md         # Этот файл

Сборка вручную (без make):
bash

nasm -f bin boot.asm -o boot.bin
nasm -f bin kernel.asm -o kernel.bin
cat boot.bin kernel.bin > os-image.bin
qemu-system-x86_64 -drive format=raw,file=os-image.bin -m 32
